❯ explain the project structure to me                                           

  Searched for 1 pattern (ctrl+o to expand)

● Here's the project structure explained:

  expense-tracker/
  │
  ├── app.py                  # Entry point — defines all URL routes and starts
  the Flask server
  │
  ├── requirements.txt        # Lists all Python dependencies (flask, pytest,
  etc.)
  ├── README.md               # Project documentation
  │
  ├── database/               # Database layer
  │   ├── __init__.py         # Makes 'database' a Python package
  │   └── db.py               # Database connection and query logic
  │
  ├── templates/              # HTML pages (rendered by Flask via Jinja2)
  │   ├── base.html           # Base layout — other pages extend this (navbar,
  footer, etc.)
  │   ├── landing.html        # Home/welcome page
  │   ├── login.html          # Login form
  │   └── register.html       # Registration form
  │
  ├── static/                 # Frontend assets (served directly to the browser)
  │   ├── css/style.css       # App-wide styles
  │   └── js/main.js          # Client-side JavaScript
  │
  └── venv/                   # Virtual environment (local Python packages — not
   committed to git)

  How it all connects:
  1. A user visits a URL → app.py matches the route
  2. The route calls render_template() → Flask loads the HTML from templates/
  3. Templates inherit from base.html for a consistent layout
  4. static/ files (CSS/JS) are loaded by the browser to style and add
  interactivity
  5. database/db.py handles all data storage and retrieval
